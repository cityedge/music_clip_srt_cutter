# Music Clip & SRT Cutter for Short Movie v1.0.0 User Guide

Music Clip & SRT Cutter for Short Movie は、フルコーラスの音源ファイルと、その曲に対応する完成済みSRT字幕から、ショート動画用のWAV音源とSRT字幕を作成するためのローカルHTMLアプリです。

このアプリは、字幕タイミングを新しく作り直すためのツールではありません。前提は、フルコーラス用SRTがすでに完成していて、歌詞表示のタイミングも信頼できる状態になっていることです。その完成済みSRTをマスターとして扱い、ショート動画化のために必要な範囲切り出し、先頭フェードイン、終端フェードアウト、終端字幕の扱いだけを調整します。

## このツールでできること

Music Clip & SRT Cutter for Short Movie では、次の作業ができます。

- MP3またはWAV音源を読み込む
- SRT字幕ファイルを読み込む
- 音源とSRTをドラッグ＆ドロップでまとめて読み込む
- SRT行をクリックして、その字幕の位置へ移動する
- 波形をクリックして、再生開始位置を選択する
- 現在位置から音声を再生する
- 再生中に同じボタンで停止する
- 停止後、再生開始位置を直前の選択位置へ戻す
- ショート音源の開始位置を手動で決める
- 開始位置の直前に短いフェードインを付ける
- 179.800秒などの安全上限を表示する
- 安全上限以内で、任意の終了点を決める
- フェードアウト開始点と終了点を2点で指定する
- フェードイン・フェードアウトを反映した状態で音声を確認する
- 終端と重なる字幕を、そのまま残す、編集して残す、または削除する
- ショート用WAVを保存する
- ショート用SRTを保存する
- 日本語 / English のUI表示を切り替える
- マウスオーバーで各項目のヒントチップを確認する

## このツールが向いている用途

このツールは、次のような作業に向いています。

- フルコーラス曲から、YouTube Shortsなどに使うショート動画用音源を作りたい
- 1番サビなど、曲中の任意の位置からショート音源を始めたい
- ショート動画の冒頭でクリックノイズを出したくない
- 3分制限を超えないように、179.800秒などの安全マージンつきで音源を切りたい
- 曲の展開に合わせて、120秒付近など3分より短い位置で自然に終わらせたい
- フェードアウトの開始点と終了点を耳で確認しながら決めたい
- 完成済みフルコーラスSRTを、ショート用SRTへ安全に変換したい
- 終端で途中までしか歌詞を使わない場合に、最後の字幕だけ編集したい

このツールは、字幕の自動生成ツールではありません。SRTを新しく作る作業には向いていません。

また、音源全体のマスタリング、ラウドネス補正、ノイズ除去、動画生成を行うツールでもありません。このアプリの目的は、完成済み素材からショート動画用の音源素材と字幕素材を切り出すことです。

## 重要な考え方

### 元SRTは信頼できるマスターとして扱う

このアプリでは、元SRTはフルコーラス用に完成された「信頼できる字幕」として扱います。

そのため、後続字幕のタイミングを個別に補正する機能はありません。ショート用SRTでは、元SRTの各時刻から実カット位置を引くことで、相対タイミングを維持します。

基本式は次の通りです。

```text
ショート用字幕時刻 = 元SRT時刻 - 実カット位置
```

この式によって、元SRTで調整済みの「歌より少し早く出る」字幕タイミングが、ショート版でも維持されます。

### 開始位置と実カット位置は違う

ユーザーが決める「開始位置」は、ショート音源を気持ちよく始めるための基準点です。多くの場合、ボーカルが入る直前、または歌い出しの直前になります。

実際の音源は、その開始位置の少し前から切り出されます。

```text
実カット位置 = 開始位置 - 先頭フェードイン長
```

たとえば、開始位置が `00:48.156`、先頭フェードインが `100ms` の場合、実カット位置は `00:48.056` になります。

出力音源では、実カット位置が `00:00.000` になります。開始位置は、出力後 `00:00.100` です。

### 先頭字幕は0秒未満なら0秒に丸める

完成済みSRTでは、歌詞字幕は歌より少し早く出るように作られていることが多くあります。そのため、ショート化したときに、最初の字幕開始時刻が0秒より前になる場合があります。

この場合、SRTにはマイナス時刻を書けないため、開始時刻を0秒に丸めます。

```text
変換後の開始時刻 = 元SRT開始時刻 - 実カット位置
出力開始時刻 = max(0, 変換後の開始時刻)
```

これは「字幕を早める」処理ではありません。むしろ、本来はショート動画開始前から出ていたはずの字幕を、動画の0秒地点まで遅らせる処理です。

### 終端は音楽的判断で決める

終端側は、自動判定ではなく、耳で確認して決めます。

ユーザーは、波形上で次の2点を指定します。

```text
フェード開始点: ここまではフル音量を維持する
終了点: ここで完全に音源を終わらせる
```

フェードアウト時間は、ユーザーが数字で指定するのではなく、2点の差分として決まります。

```text
フェード時間 = 終了点 - フェード開始点
```

短く終わらせたい場合は2点を近く置き、長く余韻を残したい場合は2点を離します。

## プライバシーとローカル処理

このツールは、単体HTMLとしてブラウザ内で動作します。

音源ファイル、SRTファイル、生成されるWAV、生成されるSRTは、利用者のブラウザ内で処理されます。このツール自体は、選択した音源や字幕をサーバーへアップロードしません。

GitHub Pagesで公開した場合でも、GitHubから配信されるのはHTMLファイルとドキュメントだけです。ユーザーが選択した音源やSRTは、ユーザーのブラウザ内で処理されます。

## 推奨環境

推奨環境は、Windows上の最新の Google Chrome または Microsoft Edge です。

このツールは、次のブラウザ機能を使います。

- File API
- Drag and Drop API
- Web Audio API
- Canvas
- Blobダウンロード

MP3入力は、ブラウザの音声デコード機能に依存します。Chrome / Edgeで再生・デコードできるMP3であれば、通常は読み込めます。

ローカルHTMLとして直接開いて使用できます。サーバーは不要です。

## 対応ファイル

### 入力

#### 音源

- MP3
- WAV

音源は、フルコーラス1曲を想定しています。通常は3分から5分程度の楽曲を想定しています。

アルバム全体や長時間ライブ音源の読み込みは想定していません。Web Audio APIは音源全体をメモリ上に展開するため、長時間ファイルではブラウザやPCのメモリ負荷が大きくなります。

#### 字幕

- SRT

SRTは、対応するフルコーラス音源用に完成済みであることを前提にしています。

### 出力

#### 音源

- 16-bit PCM WAV

出力音源はWAVのみです。MP3出力はありません。

#### 字幕

- SRT

出力SRTは、ショート音源に同期するように時刻を変換し、字幕番号を1から振り直します。

## 基本ワークフロー

### 1. 音源とSRTを読み込む

`音源を選択` でMP3/WAVを選び、`SRTを選択` で対応するSRTを選びます。

または、音源ファイルとSRTファイルを一緒にドロップ領域へドラッグ＆ドロップします。

新しい音源またはSRTを読み込むと、開始位置、終了点、フェード区間、終端字幕編集、元に戻す履歴などの作業状態はリセットされます。

### 2. SRT一覧から開始したい付近へ移動する

左ペインのSRT一覧で、ショート動画の始まりにしたい歌詞行付近をクリックします。

たとえば、1番サビの1行目からショート動画を始めたい場合は、その行をクリックします。

クリックすると、波形上の選択位置がその字幕の開始時刻へ移動します。

### 3. 波形と音声を使って開始位置を決める

波形をクリックして位置を微調整し、`再生` で音を確認します。

多くの場合、開始位置はボーカルが入る直前、または歌い出しの直前に置きます。

位置が決まったら、`現在位置を開始位置にする` を押します。

この時点で、アプリは次を計算します。

```text
実カット位置 = 開始位置 - 先頭フェードイン長
出力可能長 = min(safe limit, 元音源の残り時間)
```

画面はショート出力後時刻に切り替わり、カット後の先頭が表示されます。拡大率は自動で全体表示にはなりません。これは、フェードインが自然か確認しやすくするためです。

### 4. フェードインを確認する

`先頭へ` を押すと、カット後の先頭、つまり出力後 `00:00.000` へ移動します。

`再生` を押して、先頭フェードインが自然か確認します。

クリックノイズが出る場合は、先頭フェードインを `100ms` から `200ms` へ変える、または開始位置を少し前後に調整します。

### 5. 終了点を決める

ショート出力後時刻の波形上で、終わらせたい位置をクリックします。

3分いっぱいまで使う必要はありません。曲の展開上、120秒付近、90秒付近、サビ後すぐなどで終わらせることもできます。

位置が決まったら、`現在位置を終了点` を押します。

終了点は、safe limitを超えられません。初期値では `179.800秒` が実用上限です。

### 6. フェード開始点を決める

終了点を決めた後、フル音量を維持したい最終点へ移動し、`現在位置をフェード開始` を押します。

この2点によってフェードアウト区間が決まります。

```text
フェード開始点 〜 終了点 = フェードアウト区間
```

短くバタバタと終わらせたい場合は、フェード開始点を終了点の近くに置きます。

長く余韻を残したい場合は、フェード開始点を終了点よりかなり前に置きます。

### 7. フェードアウトを聴いて確認する

フェード開始点または、その少し前へ移動して再生します。

再生音には、現在のフェードアウト設定が反映されます。終了点に到達すると再生は停止します。

必要に応じて、フェード開始点、終了点を動かして再調整します。

### 8. 終端に重なる字幕を確認する

終了点が字幕の途中に入っている場合、右ペインの `終端に重なる字幕` に、その字幕が表示されます。

`編集前テキスト` には元SRTの字幕が表示されます。

`編集後テキスト` に、出力したい最後の字幕を入れます。

- そのまま残す場合: 編集後テキストを変更しない
- 途中まで残す場合: 必要な部分だけに編集する
- 省略表現にしたい場合: 末尾に `…` などを付ける
- 字幕を消す場合: 編集後テキストを空欄にする

### 9. WAVとSRTを保存する

`WAV保存` を押すと、現在の開始・終了・フェード設定でショート音源を保存します。

`SRT保存` を押すと、現在の開始・終了・終端字幕設定でショート用SRTを保存します。

ブラウザの複数ファイルダウンロード確認を避けるため、WAVとSRTは個別の保存ボタンになっています。

## 音源とSRTの読み込み

### 通常のファイル選択

`音源を選択` でMP3/WAVを読み込みます。

`SRTを選択` でSRTを読み込みます。

ファイル選択欄はボタン形式です。ブラウザ標準の「選択されていません」表示は出ません。読み込まれたファイル名や状態は、下のステータス欄に表示されます。

### ドラッグ＆ドロップ

音源ファイルとSRTファイルを、まとめてドロップ領域へドラッグ＆ドロップできます。

```text
MP3/WAV + SRT を同時ドロップ
→ 両方読み込み

MP3/WAVだけドロップ
→ 音源だけ読み込み

SRTだけドロップ
→ SRTだけ読み込み
```

ブラウザのセキュリティ制約により、音源だけをドロップして、同じフォルダ内の同名SRTを自動検索することはできません。同時に読み込みたい場合は、音源とSRTの両方を選択またはドロップしてください。

## 波形表示と選択位置

このアプリでは、`選択位置` が非常に重要です。

再生は常に、最後に選択した位置から始まります。

再生して停止すると、再生カーソルは進んだ位置ではなく、直前の選択位置へ戻ります。

これは、同じ場所を何度も聴いて開始点や終端を調整しやすくするためです。

### 選択位置を移動する方法

選択位置は次の操作で移動します。

- 波形をクリックする
- SRT行をクリックする
- `先頭へ` を押す
- `終端へ` を押す
- キーボードの左右キーで移動する

### SRT一覧との同期

波形をクリックすると、SRT一覧も対応する字幕へ移動します。

字幕と字幕の間をクリックした場合は、直前の字幕が選ばれます。

再生中は、SRT一覧が再生カーソルに追従します。

停止すると、SRT一覧は選択位置に戻ります。

## 開始位置の決め方

### 開始位置とは

開始位置は、ショート音源を実質的に始めたい基準点です。

多くの場合、ボーカルが入るまさに直前、またはサビの歌い出し直前に置きます。

ただし、音源ファイル自体は、この開始位置ぴったりから始まるわけではありません。クリックノイズを避けるため、開始位置の少し前から切り出し、その区間にフェードインをかけます。

### 実カット位置

実カット位置は次のように決まります。

```text
実カット位置 = 開始位置 - 先頭フェードイン長
```

たとえば、次の場合を考えます。

```text
開始位置: 00:48.156
先頭フェードイン: 100ms
```

実カット位置は次のようになります。

```text
実カット位置: 00:48.056
```

出力WAVでは、`00:48.056` がショート音源の `00:00.000` になります。

### 先頭フェードイン

先頭フェードインは、演出的なフェードインではなく、クリックノイズを消すための短いフェードです。

選択肢は次の通りです。

- 50ms
- 100ms
- 200ms

通常は100msで十分です。

アタックを強く残したい場合は50ms、クリックノイズが気になる場合は200msを試してください。

### 開始後の表示

`現在位置を開始位置にする` を押すと、画面はショート出力後時刻に切り替わります。

このとき、表示は全体表示にはなりません。拡大率を維持したまま、カット後の先頭が表示されます。

これは、先頭フェードインが自然かすぐ確認できるようにするためです。

---

## 終端位置とフェードアウトの決め方

### safe limit

safe limitは、ショート動画の3分制限を超えないための実用上限です。

初期値は次の通りです。

```text
179.800秒
```

3分ちょうどの180.000秒を狙うと、動画生成やエンコードの過程でわずかに超える可能性があります。そのため、このアプリでは少し余裕を持った179.800秒を初期値にしています。

safe limitは右ペインの `safe limit（秒）` で変更できます。

### 終了点

終了点は、ショート音源が完全に終わる位置です。

safe limit以内であれば、任意の位置に置けます。

たとえば、曲の展開上、120秒で終わらせたい場合は、120秒付近に移動して `現在位置を終了点` を押します。

### フェード開始点

フェード開始点は、フル音量を維持したい最終点です。

ここから終了点まで、フェードアウトがかかります。

### フェード時間

フェード時間は直接入力しません。

次の差分として自動的に決まります。

```text
フェード時間 = 終了点 - フェード開始点
```

### マーカーのドラッグ調整

波形上で、フェード開始点や終了点のマーカー付近をドラッグすると、位置を直接調整できます。

短いフェードにしたい場合は、フェード開始点を終了点の近くへ動かします。

長いフェードにしたい場合は、フェード開始点を左へ動かします。

---

## 終端に重なる字幕の処理

終了点が字幕の途中に入る場合、その字幕は `終端に重なる字幕` として表示されます。

たとえば、元SRTに次の字幕があるとします。

```text
02:57.000 --> 03:01.000
あなたといた あの時間
```

出力音源が `02:59.800` で終わる場合、この字幕は終了点をまたぎます。

このとき、アプリは右ペインにその字幕を表示します。

### そのまま残す

編集後テキストを変更しなければ、字幕テキストはそのまま出力されます。

ただし、終了時刻は音源終了点に丸められます。

```text
02:57.000 --> 02:59.800
あなたといた あの時間
```

### 編集して残す

途中まで残したい場合は、編集後テキストを修正します。

例:

```text
あなたといた
```

または、途中で切れる感じを出したい場合は、次のようにできます。

```text
あなたといた…
```

### 消す

編集後テキストを空欄にすると、その字幕は出力されません。

終端字幕処理は、プレビュー範囲を変更するものではありません。あくまで、最終出力SRTでその字幕をどう扱うかを指定するための設定です。

---

## 音声プレビュー

音声プレビューは、現在の編集状態を反映します。

開始位置設定後は、次の処理が反映された仮想ショート音源として再生されます。

- 実カット位置から再生
- 先頭フェードイン
- フェードアウト
- 終了点で停止

### 再生と停止

`再生` を押すと、選択位置から再生します。

再生中は、同じボタンが `停止` に変わります。

`停止` を押すと再生が止まり、再生カーソルは選択位置へ戻ります。

### 終了点到達時

出力後モードで再生している場合、終了点に到達すると自動停止します。

元音源がその後も続いていても、出力音源上では終了点で終わるため、それ以降は再生されません。

---

## 出力

### WAV保存

`WAV保存` を押すと、現在の設定でショート用WAVを保存します。

出力ファイル名は、おおむね次の形式です。

```text
元音源ファイル名_short.wav
```

出力WAVは16-bit PCM WAVです。

### SRT保存

`SRT保存` を押すと、現在の設定でショート用SRTを保存します。

出力ファイル名は、おおむね次の形式です。

```text
元SRTファイル名_short.srt
```

SRTファイルが未読込の場合は保存できません。

### WAVとSRTを別々に保存する理由

ブラウザでは、複数ファイルを連続ダウンロードしようとすると、確認ダイアログが出ることがあります。

そのため、このアプリではWAVとSRTを同時保存するボタンは用意せず、個別の保存ボタンにしています。

## 言語切り替えとヒントチップ

画面右上の `言語／Language` で、日本語とEnglishを切り替えられます。

切り替え対象は、主要なUIテキスト、ボタン、ラベル、状態表示、ヒントチップです。

各ボタンや入力欄にマウスを重ねると、ヒントチップが表示されます。

操作に迷った場合は、まず対象のボタンや欄にマウスを置いて説明を確認してください。

## 用語解説

### 元音源時刻

読み込んだフルコーラス音源上の時刻です。

### ショート出力後時刻

ショート用WAVとして切り出した後の時刻です。

実カット位置が、ショート出力後の `00:00.000` になります。

### 選択位置

次の再生開始位置です。

波形クリック、SRT行クリック、先頭へ、終端へ、キーボード操作などで移動します。

### 再生カーソル

再生中に実際に進んでいる位置です。

停止すると、再生カーソルは選択位置へ戻ります。

### 開始位置

ショート音源を実質的に始めたい基準点です。

多くの場合、ボーカルが入る直前に置きます。

### 実カット位置

実際に音源を書き出し始める位置です。

開始位置より、先頭フェードイン長だけ前になります。

### 先頭フェードイン

実カット位置から開始位置までの短いフェードインです。

クリックノイズを避ける目的で使います。

### safe limit

ショート動画の3分制限を超えないための実用上限です。

初期値は179.800秒です。

### 終了点

出力音源が完全に終わる位置です。

### フェード開始点

ここまではフル音量を維持し、ここから終了点まで音量を下げます。

### 終端に重なる字幕

終了点をまたぐ字幕です。

そのまま残す、編集して残す、空欄にして消す、という扱いを選べます。

---

## 注意点と制限

- このアプリは、完成済みSRTを信頼してショート用に変換するツールです。壊れたSRTの修復や、字幕タイミングの自動補正は目的としていません。
- 音源だけを読み込んで、同じフォルダ内の同名SRTを自動検索することはできません。ブラウザのセキュリティ仕様上、ユーザーが渡していないファイルにはアクセスできないためです。
- MP3入力はブラウザのデコード機能に依存します。
- 出力音源はWAVのみです。MP3出力はありません。
- 出力WAVは16-bit PCM WAVです。
- 長時間音源や大量ファイルの処理は想定していません。
- フェードイン・フェードアウトは音量処理です。元音源の音楽構成を解析して自動で最適化するものではありません。
- 終端に重なる字幕が複数あるケースは通常想定していません。万一複数ある場合、終端に近い1件だけを編集対象にします。
- 動画ファイルは生成しません。動画化は別ツールで行ってください。
- safe limitを180.000秒に近づけると、動画生成後に3分をわずかに超える可能性があります。

## トラブルシューティング

### 音源が読み込めない

MP3またはWAVか確認してください。

MP3の場合、ブラウザがそのファイルをデコードできないことがあります。その場合は、別の形式で書き出すか、WAVに変換してから読み込んでください。

### SRTが読み込めない

SRTフォーマットになっているか確認してください。

このアプリは、完成済みSRTを前提にしているため、極端に壊れたタイムスタンプや、特殊な字幕形式には対応していません。

### ドロップしても同名SRTが自動で読まれない

通常のHTML5アプリでは、音源ファイルだけをドロップして、同じフォルダのSRTを勝手に探すことはできません。

音源とSRTを同時にドロップしてください。

### 字幕が少し遅いように感じる

まず、元のフルコーラスSRTが意図通りに早め表示されているか確認してください。

このアプリは、後続字幕のタイミングを個別に補正しません。後続字幕が遅い場合は、元SRT側の修正が必要です。

ただし、最初の字幕だけは、0秒未満になる場合に0秒へ丸められます。

### 再生を停止するとカーソルが戻る

これは仕様です。

このアプリでは、選択位置を固定したまま同じ場所を何度も聴く操作を重視しています。停止すると、再生カーソルは直前の選択位置へ戻ります。

### 先頭へ・終端へを押しても拡大率が変わらない

これは仕様です。

拡大率が変わるのは、`縮小`、`標準倍率`、`拡大`、`全体表示` を押した場合だけです。

### 最後の字幕を消したい

`終端に重なる字幕` の `編集後テキスト` を空欄にしてください。

空欄にすると、その字幕は出力SRTに含まれません。

### WAVとSRTを同時に保存したい

このアプリでは、WAVとSRTを別々のボタンで保存します。

ブラウザによっては、複数ファイルを連続ダウンロードすると確認ダイアログが出るためです。
